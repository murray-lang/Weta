PARAM

VM Stack
BeforeAfter
  
variable address

Abstract

Push the address of the given parameter onto the stack. Currently the assembler only accepts the label/name of a parameter.

The symbol is resolved by the assembler into an offset from the beginning of the parameters on the call frame of the current procedure. From this the firmware calculates an absolute address, and pushes that onto the stack. Thus any subsequent references to the address on the stack need not be concerned about its scope.

Note that local works in a similar fashion.

Synopsis

param    <symbol>

Arguments

<symbol> - name of a procedure parameter

Returns

Returns a variable address (by virtue of it being pushed onto the stack)
