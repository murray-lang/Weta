SET

VM Stack
BeforeAfter
  
var-address
value

Abstract

Place the value second from the top of the stack into the location found at the top of the stack.

The data type expected by set is determined by the last with<type> instruction. The valid options for set are: withbool, withint8, withuint8, withint16, withuint16, withint32, withuint32, withfloat, withdouble, withptr and withstring.

An alternative to explicitly calling one of the with<type> instructions before calling set is to use one of the prefixed versions of the the instruction, namely:

qset (bool), bset, ubset, sset, usset, iset, uiset, fset and dset, pset and strset.

The assembler will expand these to a corresponding with<type> instruction followed by set.

Synopsis

<value>

<var-address>

set

Arguments

<value> - the value to be assigned to the variable

<var-address> - the address of the variable to set

    Returns

    Nil
