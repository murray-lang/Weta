RETURN

VM Stack
BeforeAfter
  
caller's context
return address

Abstract

return causes execution to return to a procedure's calling context. Before doing so, it unwinds the local frame and restores the machine context of the caller, both of which were set up and saved respectively by begin.

While it operates on the VM stack to restore this information, this will be specific to the firmware implementation and is not relevant for the purpose of the general processing instructions (other than being prone to corruption by unbalanced stack operations).

Synopsis

return

Arguments

None

Returns

Nothing is returned to the stack for general use.

Example

        myBeep  .proc
            begin
            beep
            return
            .endproc
