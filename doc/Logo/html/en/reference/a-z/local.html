LOCAL

Declare a variable to be in procedure-local scope. In this version of Logo, variables first used in a procedure are local to that procedure anyway. local was included for compatibility reasons, but it can also be employed to mitigate against accidental name clashes with global variables.

Synopsis

local <name>

Arguments

<name> - name of the variable

    Returns

    Nil.

    Example

              ...
          make "foo 42      ; A global variable
          ...
          exit

          to myProc
            local bar        ; bar is not visible to global scope
            make "fred 3     ; Actually, neither is fred
            make "bar 54
            output :bar
          end
