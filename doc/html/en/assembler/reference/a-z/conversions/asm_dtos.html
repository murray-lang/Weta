DTOS

VM Stack
BeforeAfter
  
double valueuint16 value

Abstract

Pop the double from the stack and push a uint16 (unsigned short) with the same value.

Note

It is assumed that the programmer understands the possibility of overflow in this situation. This instruction is a necessary part of the language specification, but there is little that can be done in the assembler to mitigate the possibility. The programmer is responsible for ensuring that values are within the range that can be represented by the narrower data type. The following discussion assumes no loss of data due to overflow.

What is placed on the stack is an unsigned short, but the double removed from the stack is first cast to a signed short (int16) so, assuming no overflow, the sign bit of the value will be extended to the full width of the short. The fact that it is then pushed (cast) as an unsigned short makes no difference to its contents. If it is later interpreted as an int16 (ie. cast as an int16 when popped off the stack) then the original signed value will be returned.

This is why there are not separate instructions for converting to signed values and to unsigned values. To have these would use up a large number of the base VM op-codes for no benefit. It is arguable that the name of the instruction should have been dtous, as this reflects the literal truth, but it would also be misleading because it suggests that sign is lost. Treat the 's' as a generic abbreviation for short, signed or otherwise.

Synopsis

<float>    <value>

dtos

Where

<value> - the double value to be converted

Note

This synopsis shows an immediate value being explicitly placed onto the stack to be converted. However, the value could be the result peripheral I/O, function calls or any number of mathematical expressions. Or it could be an arbitrary combination of all of these.

Returns

Returns a uint16 with the value of <value> (by virtue of it being pushed onto the stack). It can be treated as an int16 by subsequent code.

Example

        ...
        double   -42
        dtos
        withint16    ; treat everything as an int16 (signed)
        tostr       ; convert signed value to a string ("-42")
        withstring
        send        ; send "-42" to the default stream
        ...
