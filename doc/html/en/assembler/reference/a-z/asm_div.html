Abstract

Remove the two top numerical values from the stack, and push the the value calculated by dividing the second top value by the top value.

The data type expected by mul is determined by the last with<type> instruction. The valid options for div are: withint8, withuint8, withint16, withuint16, withint32, withuint32, withfloat and withdouble.

An alternative to explicitly calling one of the with<type> instructions before calling div is to use one of the prefixed versions of the the instruction, namely:

bdiv, ubdiv, sdiv, usdiv, idiv, uidiv, fdiv and ddiv

The assembler will expand these to a corresponding with<type> instruction followed by div.

Synopsis

value-1

value-2

mul

Arguments

<value-1> - the first argument (left hand arg from higher level language)

<value-2> - the second argument (right hand arg from higher level language)

    Returns

    Returns <value-1> divided by <value-2>.
