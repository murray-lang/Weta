MATH.FLOOR

VM Stack
BeforeAfter
  
valuevalue rounded down

Abstract

Pop the value from the top of the stack, and push the value rounded down to the nearest lower whole number. (i.e. the floor). If there is no fractional part then the original value is pushed.

The data type expected by math.floor is determined by the last with<type> instruction. The valid options for math.floor are: withfloat and withdouble.

An alternative to explicitly calling one of the with<type> instructions before calling math.floor is to use one of the prefixed versions of the the instruction, namely:

math.ffloor or math.dfloor

The assembler will expand these to a corresponding with<type> instruction followed by math.floor.

Synopsis

value

math.floor

Arguments

<value> - the value to round down

    Note

    This synopsis shows an immediate value being explicitly placed onto the stack. However, the value could be the result peripheral I/O, function calls or any number of mathematical expressions. Or it could be an arbitrary combination of all of these.

    Returns

    Returns the value rounded down to the nearest lower whole number. If there is no fractional part then the original value is returned.
