GET

VM Stack
BeforeAfter
  
var-addressvalue

Abstract

Pop the variable address from the top of the stack and push the contents of that variable.

The data type expected by get is determined by the last with<type> instruction. The valid options for get are: withbool, withint8, withuint8, withint16, withuint16, withint32, withuint32, withfloat, withdouble, withptr and withstring.

An alternative to explicitly calling one of the with<type> instructions before calling get is to use one of the prefixed versions of the the instruction, namely:

qget (bool), bget, ubget, sget, usget, iget, uiget, fget and dget, iget, uiget, fget and dget, pget and strget.

The assembler will expand these to a corresponding with<type> instruction followed by get.

Synopsis

<var-address>

get

Arguments

<var-address> - the address of the variable to get the contents of

    Returns

    Returns the contents at the given variable address (ie. it is pushed onto the stack)
